# Build-Anker

Nimm einen öffentlichen CVE-Eintrag (Sicherheitslücken-Datenbank, kostenlos zugänglich), lass Claude den zugehörigen Quellcode-Stand vor dem Fix via GitHub abrufen, und miss mit zwei verschiedenen Modellen (z. B. Claude Haiku vs. GPT-4o mini), ob sie die Lücke blind benennen – so siehst du selbst, wie gross die Lücke zwischen Modellen bei echten Sicherheitsbugs ist.
